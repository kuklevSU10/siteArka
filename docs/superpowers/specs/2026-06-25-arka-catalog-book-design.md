# АРКА — интерактивная книга-каталог · дизайн-спека

Дата: 2026-06-25
Статус: на ревью
Источник истины по бренду: `Arka/BRANDBOOK.md` (v0.1)
Репозиторий реализации: `siteArka` (git, GitHub Pages)

## 1. Цель и рамки

После hero-заставки добавить секцию с **интерактивной книгой-каталогом работ**: лежащая закрытая книга, клик открывает её в разворот, страницы листаются по клику/свайпу. Содержимое: 3 интерьерных проекта по 4 кадра (12 готовых снимков в `assets/interiors/`). Эстетика: матовая печатная **монография / архитектурный журнал** (референсы `photo_2026-06-16` и `photo_2026-06-25`), адаптированная под бренд АРКА, не клон.

Остаёмся на чистом HTML/CSS/JS без сборщика и бэкенда. Книга = самостоятельный модуль, встроенный в существующий лендинг.

Принцип «лучшести»: книга как физический объект + крупная типографика-номер + редакционный ритм. Сдержанность как подпись (приглушаем тени StPageFlip до матовости).

## 2. Зафиксированные решения

- **Размещение:** секция `#works` сразу после hero, до `#about`. Пункт меню «Работы». Книга открывается из закрытой обложки **внутри секции** (не полноэкранно).
- **Механика:** StPageFlip (`page-flip`, MIT), self-host UMD-дистрибутива в `vendor/page-flip/`. HTML-страницы. Тени/текстура приглушены под матовый референс.
- **Обложка:** терракотовая, белый/золотой вордмарк и арка с эффектом тиснения (как референс).
- **Фирменные акценты (все включены):**
  1. двойной кадр через корешок (один раз, кульминация);
  2. арка-вырубка (арочная маска фото) на опенерах глав;
  3. манжетка (belly band) с тезисом поперёк закрытой обложки;
  4. звук перелистывания (тумблер, по умолчанию выкл).
- **Палитра-на-главу:** цвет страницы-опенера сэмплируется из доминирующего тона фото проекта (studio / classic / burgundy).

## 4. Структура книги (модель страниц)

StPageFlip с `showCover: true`; обложки и задняя крышка помечаются `data-density="hard"`. Пропорция страницы около 1:1.15 (квадратное фото 1:1 + поле под подпись/раннеры). Разворот = 2 страницы.

**Инвариант учёта фото:** у каждого проекта ровно 4 кадра, каждый показан ровно один раз. Один из 4 = «герой», остальные 3 = галерея. Главы 01 и 02 занимают по 2 разворота (опенер + галерея). Глава 03 = кульминация, 3 разворота (опенер + двойной кадр + пара). Итого 20 страниц (чётно для StPageFlip).

Арка-вырубка применяется к кадру-герою на опенерах глав 01 и 02 (одиночная страница). Двойной кадр через корешок = одноразовый приём, отдан главе 03.

Порядок страниц:

| # стр. | Тип | Лево | Право | Кадров |
|---|---|---|---|---|
| 1 | Обложка (hard) | (торец) | терракота, вордмарк, арка, «Каталог» | 0 |
| 2-3 | Содержание | «Каталог · 2026» + интро-строка (тон брендбука §13) | список 01 Studio / 02 Classic / 03 Burgundy + номера, кликабельно | 0 |
| 4-5 | Глава 01 опенер | greige: призрак-номер `01`, название, абзац, свотчи палитры | фото-герой в арке-вырубке + подпись + hairline | 1 |
| 6-7 | 01 галерея (1 крупный + 2 стопкой) | крупный кадр | 2 кадра, тонкие зазоры | 3 |
| 8-9 | Глава 02 опенер | cream, `02`, название, текст, свотчи | фото-герой в арке-вырубке + подпись | 1 |
| 10-11 | 02 галерея (3-up лента) | кадр | 2 кадра | 3 |
| 12-13 | Глава 03 опенер | терракота (тёмный момент), `03`, warm-white текст | фото-герой в арке-вырубке + подпись | 1 |
| 14-15 | 03 двойной кадр | левая половина снимка | правая половина (через корешок) | 1 |
| 16-17 | 03 пара | кадр + подпись-колонка | кадр | 2 |
| 18-19 | Финал | бренд-цитата Cormorant | CTA «Обсудить проект» → ссылка на `#contact` | 0 |
| 20 | Задняя обложка (hard) | мини-вордмарк + арка-знак | (торец) | 0 |

Сумма: 4 + 4 + 4 = 12 кадров, каждый один раз. Точное распределение «какой из 4 кадров герой» закрепляется при реализации после просмотра кадров проекта.

## 5. Книжная фурнитура (из референса, под бренд)

- Раннеры сверху: `АРКА \\ Каталог` (слева), `Избранные интерьеры \\ <раздел>` (справа). Manrope, мелкий, разрядка, muted. ASCII `\\`, без типографских тире.
- Номера страниц в нижних внешних углах.
- Тонкий вертикальный hairline у корешка на цветных страницах.
- Призрачные номера глав: Cormorant, крупные, тонально-в-тон, обрезаны краем страницы.
- Подписи комнат по-русски из имён файлов: Кухня-гостиная, Спальня, Ванная, Детская, Гардероб, Душевая, Гостиная-спальня и т.п.
- Шрифты бренда: Cormorant в заголовках/номерах/цитатах, Manrope в раннерах/подписях/тексте.

## 6. Презентация-объект (стейдж)

- Книга лежит на `--book-stage` поверхности; многослойная мягкая контактная тень.
- В покое (закрыта) лёгкий наклон/перспектива + видимый торец (стопка кремовых страниц справа) = «толстый каталог». Манжетка поперёк обложки с тезисом.
- По клику: манжетка уезжает, книга выпрямляется, чуть «поднимается» (scale) и центрируется, StPageFlip раскрывается.
- Матовость: зерно бумаги (очень слабое), едва заметный световой градиент поперёк разворота; тени StPageFlip приглушены (`maxShadowOpacity` низкий, без глянца/текстуры).

## 7. Данные и компоненты (изоляция)

- **`book-data.js`** (или константа в `book.js`): массив
  `PROJECTS = [{ number, name, subtitle, description, accent, pageBg, hero:{src,caption}, photos:[{src,caption}] }]`.
  Реальные названия/тексты/фото меняются только здесь.
- **`book.js`** (новый модуль, defer): чистые функции
  - `buildPages(PROJECTS)` → собирает DOM страниц (обложка, содержание, главы, финал, задняя обложка) внутрь `#arka-book`;
  - `initFlip()` → монтирует StPageFlip по клику на закрытую книгу; настройки матовости и портретного режима;
  - `wireNav()` → клик-половины, стрелки, клавиши ← →, свайп, индикатор «N / M», закрытие;
  - `wireLazy()` → жадно текущий+соседний разворот, остальное `lazy`; на событии `flip` подгружает соседей;
  - `wireSound()` → тумблер шелеста (localStorage, по умолчанию выкл).
  Каждая функция тестируема изолированно; `book.js` не трогает остальной сайт.
- **`tools/optimize-interiors.js`** (sharp): `assets/interiors/_src/*.png` → `assets/interiors/*-800.webp`, `*-1254.webp`. Запускается один раз локально, коммитятся только WebP. Исходные PNG перемещаются в `assets/interiors/_src/` и добавляются в `.gitignore` (не коммитятся, не уезжают в прод; деплой грузит `path: .`, поэтому всё в репозитории попадает в прод).
- **`tools/sample-palette.js`** (sharp): доминирующий тон каждого проекта → значения `--book-ch1/2`, `--book-terracotta` (разовый помощник; значения вшиваются в CSS).

## 8. Взаимодействие и доступность

- Открытие: клик/Enter по закрытой книге (`role="button"`, `tabindex`, `aria-label`).
- Листание: клик по правой/левой половине; тонкие золотые стрелки; клавиши ← →; свайп (тач); индикатор страниц.
- Содержание и финальный CTA кликабельны (переход к главе / к `#contact`).
- Закрытие: кнопка «Закрыть» возвращает к закрытой обложке.
- `prefers-reduced-motion`: короткий кросс-фейд вместо сгиба, без «подъёма»/наклона.
- `<noscript>`: статическая адаптивная сетка 12 фото по проектам (контент и SEO без JS).
- Видимый keyboard-focus (золотой outline) на интерактивных элементах.

## 9. Производительность

- 12 PNG (~28 МБ) → WebP на 2 ширинах (~150-350 КБ каждое, итого ~3-4 МБ).
- WebP без PNG-фолбэка (поддержка ~97%); `srcset` 800/1254 + `sizes` под размер страницы. noscript-сетка тоже на WebP.
- Ленивая загрузка: cover + первый разворот жадно; остальное `loading="lazy"` + предзагрузка соседей по событию `flip`.
- StPageFlip и звук грузятся только когда секция в зоне видимости (IntersectionObserver), чтобы не тормозить hero.

## 12. Вне рамок / открытые входы

- Реальные названия проектов и описания (сейчас рабочие плейсхолдеры studio/classic/burgundy в тоне брендбука) — подмена в `book-data.js` позже.
- Файл звука перелистывания (короткий шелест) — подобрать лицензионно чистый или сгенерировать; до этого тумблер присутствует, звук опционален.
- Полноэкранный режим книги — возможное v1.1, сейчас вне рамок.
- Версионирование: коммитим в `siteArka`, деплой на Pages при пуше в `main`.

## 13. Критерии приёмки

- Книга читается как физический матовый объект (тень, торец, наклон), не как веб-виджет.
- Перелистывание плавное по клику/свайпу/клавишам; корректный одностраничный режим на мобиле.
- Все 12 кадров показаны, каждый один раз; подписи корректны.
- Палитры глав рифмуются с фото; терракотовая обложка как референс; арка-вырубка и двойной кадр присутствуют.
- Суммарный вес кадров ~3-4 МБ; hero не тормозит (книга инициализируется по видимости).
- `node test-site.js` → `Arka site static checks passed.`; нет горизонтального скролла и перекрытий на 1440/768/390.
- Формула: «дорого, тихо, архитектурно, точно».
